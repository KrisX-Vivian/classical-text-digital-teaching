基于【00_通用设计规则.txt】要求和index.html风格，优化 resource.html 资源库页面，仅修改样式、动效与内容展示形式，不改动现有资源分类逻辑、功能入口及代码注释，新增 CSS 样式存入 assets/css/，视觉风格贴合国风，同时突出资源库的实用性与便捷性，做到 “美用兼顾”。
一、视觉设计精细化要求
整体布局：沿用现有资源分类框架，采用「筛选导航 + 卡片式资源展示」布局，栅格化对齐，核心内容区留白≥30%；筛选导航区背景为淡墨浅灰（#E8E5E1），与资源展示区形成视觉区分，不改动现有筛选分类维度。
筛选导航：
样式：国风导航按钮，米白背景 + 赭石棕细边框，选中状态为朱砂红填充 + 白字，按钮文字为思源宋体，14-16px；
图标：筛选分类图标为国风线性款（1.5px 线条），融合书卷 / 竹简元素，色调为赭石棕，hover 时变为朱砂红。
资源卡片：
基础样式：统一比例裁剪（3:2），米白宣纸背景 + 赭石棕 1px 细边框，卡片内包含「资源封面图 + 资源名称 + 资源类型 + 简介」，文字按全局字体层级规范排布，资源名称用二级标题，简介用正文，资源类型用朱砂红小标签标注；
图片处理：资源封面图存入 assets/images/，做「去噪 + 色彩校准 + 国风色调调色 + 轻微锐化」处理，采用渐进式加载（先模糊水墨缩略图再清晰展示）；核心资源封面图添加「赭石棕细边框 + 轻微水墨投影」（模糊半径 8px，不透明度 10%）；
动效：卡片 hover 时轻微上浮（y 轴偏移 5px）+ 墨灰阴影加深，滚动到可视区域时水墨淡入 + 轻微上移动效（时长 0.5s），批量资源卡片按列依次延迟 0.1s 出现。
视觉装饰：页面装饰以「低透明度古籍纹理 + 竹简横线抽象纹理」为主，占比≤页面 5%，仅在筛选导航区、页面边角做轻量装饰，不干扰资源内容阅读；禁用复杂水墨元素，保证页面简洁实用。
二、交互体验精细化要求
基础交互：沿用全局导航、按钮、滚动交互规则，所有按钮（资源查看、下载）为国风圆角样式（圆角 8px），贴合全局交互色彩与动效要求。
资源筛选交互：筛选按钮点击时无卡顿，选中状态视觉反馈清晰；支持多条件筛选，筛选结果实时刷新，刷新时添加「轻量水墨加载动画」（竹简纹路旋转），避免空白等待。
资源浏览交互：
长页面设置「书卷造型回到顶部按钮」，滚动超过 500px 时淡入，贴合全局样式；
资源简介 Hover 时显示完整内容（朱砂红边框提示框），提示框为米白背景，不遮挡其他资源卡片。
资源下载交互：点击下载按钮后，显示「国风轻量反馈弹窗」（无遮罩、居中、书卷边框），提示语如 “资源下载成功”，弹窗 3 秒后水墨淡隐；下载失败时，错误项边框为朱砂红，下方显示朱砂红小字提示，无刺耳提示音。
三、内容呈现要求
资源标注：所有资源标注清晰的「类型 + 适用场景 + 更新时间」，用淡墨灰辅助文字呈现，与资源名称形成视觉层级；
关键词强化：资源名称中的核心文本名、作者名用赭石棕加粗，方便用户快速识别；
排版规范：资源简介文字行间距 1.6 倍，字数控制在 50 字内，多余内容用「...」省略，hover 时展示完整内容。
