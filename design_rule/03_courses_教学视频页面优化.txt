基于【00_通用设计规则.txt】要求，优化 courses.html 教学视频页面，仅修改样式、动效与视频展示形式，不改动现有视频分类逻辑、播放入口及代码注释，新增 CSS 样式存入 assets/css/，视频素材存入 assets/videos/（MP4 格式），视觉风格贴合国风，同时保证视频播放的便捷性与流畅性，适配多终端播放。
一、视觉设计精细化要求
整体布局：沿用现有视频分类框架，采用「分类导航 + 视频列表 / 封面展示」布局，Bootstrap12 列栅格对齐，核心内容区留白≥30%；分类导航区与视频展示区用赭石棕 1px 细线条分隔，视觉层次清晰，不改动现有视频分类维度（如文本类型、教学阶段）。
分类导航：
样式：与资源库页面筛选导航风格统一，国风按钮（米白背景 + 赭石棕细边框），选中状态朱砂红填充 + 白字，文字为思源宋体 14-16px；
图标：分类图标融合「书卷 + 播放按钮」国风线性元素（1.5px 线条），赭石棕色调，hover 时变为朱砂红。
视频展示模块：
视频封面：统一比例裁剪（16:9，符合视频播放规范），存入 assets/images/，做国风色调调色处理，封面添加「极简赭石棕播放按钮图标」（居中显示），hover 时播放按钮变为朱砂红且轻微放大（1.1 倍）；
视频卡片：包含「视频封面 + 视频标题 + 主讲人 + 时长 + 简介」，卡片为米白宣纸背景 + 赭石棕细边框，文字按全局字体层级排布，视频标题用二级标题（赭石棕），主讲人 / 时长用朱砂红小标签标注，简介用墨灰正文；
动效：视频卡片 hover 时轻微上浮（y 轴偏移 5px）+ 墨灰阴影加深，滚动到可视区域时水墨淡入 + 轻微上移动效（时长 0.5s），批量卡片依次延迟 0.1s 出现。
视频播放区：
样式：播放区背景为墨灰（#363636），四周留白≥20%，播放控件做国风优化（进度条为赭石棕，暂停 / 播放按钮为朱砂红国风造型）；
装饰：播放区边角添加轻量竹简纹装饰，占比≤播放区 5%，不干扰视频观看。
二、交互体验精细化要求
基础交互：沿用全局导航、按钮、滚动交互规则，所有功能按钮（播放、收藏、查看详情）为国风圆角样式（圆角 8px），贴合全局交互色彩与动效要求。
视频分类交互：分类按钮点击反馈清晰，选中状态视觉区分明显；支持多条件分类，分类结果实时刷新，刷新时添加「竹简纹路旋转」水墨加载动画。
视频播放交互：
封面点击直接播放视频，支持暂停 / 全屏 / 进度调节，全屏播放时控件适配屏幕尺寸；
视频自动静音播放，播放页提供音量调节按钮，音量按钮为国风线性样式；
移动端视频播放适配竖屏 / 横屏，横屏播放时隐藏非核心导航元素，保证播放视野。
收藏 / 点赞交互：收藏 / 点赞按钮为国风图标（书卷 / 小红花造型），默认赭石棕，点击后变为朱砂红，点击后显示「国风轻量反馈弹窗」（如 “视频收藏成功”），3 秒后水墨淡隐。
三、多媒体与内容要求
视频规范：教学视频样本为 MP4 格式，存入 assets/videos/，时长≤10 分钟，保证播放流畅；视频添加字幕，字幕字体为思源宋体，颜色为白字 + 墨灰描边，保证观看清晰。
内容呈现：
视频标题提炼核心教学内容，关键词用朱砂红加粗；
主讲人 / 时长标注在视频封面下方，位置统一，视觉规范；
视频简介控制在 80 字内，行间距 1.6 倍，墨灰正文，突出教学重点与适用人群。
性能优化：视频采用「懒加载」模式，滚动到可视区域时再加载视频资源，避免页面加载过慢；移动端自动降低视频分辨率，保证播放流畅。
