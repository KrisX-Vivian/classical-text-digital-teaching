基于【00_通用设计规则.txt】要求，优化 exercises.html 题库 + AI 工具页面，仅修改样式、动效与功能展示形式，不改动现有题库分类、AI 工具入口及代码注释，新增 CSS 样式存入 assets/css/，贴合「古典文本数字化」特色，兼顾题库的实用性与 AI 工具的交互性，做到 “功能清晰、视觉统一”。
一、视觉设计精细化要求
整体布局：沿用现有「题库模块 + AI 工具模块」双分区框架，采用 Bootstrap12 列栅格布局，两分区用赭石棕 1px 细线条 + 留白分隔，核心内容区留白≥30%；题库区偏实用简洁，AI 工具区可适当增加国风数字交互元素，不改动现有模块顺序与功能入口。
题库模块：
筛选导航：与资源库、课程页导航风格统一，国风按钮（米白背景 + 赭石棕细边框），选中状态朱砂红填充 + 白字，筛选维度（难度 / 文本类型 / 题型）保留现有逻辑，图标融合「竹简 + 问号」国风线性元素；
题库卡片：统一比例（4:3），米白宣纸背景 + 赭石棕细边框，包含「习题名称 + 难度 + 文本类型 + 答题入口」，难度用朱砂红星级标注（★），文本类型用赭石棕小标签标注，文字按全局字体层级排布；
动效：卡片 hover 时轻微上浮 + 墨灰阴影加深，滚动到可视区域时水墨淡入 + 轻微上移动效，批量卡片依次延迟 0.1s 出现；答题按钮为国风圆角样式，贴合全局交互规则。
AI 工具模块：
工具入口：采用「国风卡片 + 功能图标」形式，卡片为书卷造型，包含「工具图标 + 工具名称 + 功能简介 + 使用入口」，图标融合古典元素（毛笔 / 批注）与数字元素（代码 / 箭头），1.5px 线性国风款，赭石棕色调；
交互界面：AI 工具使用界面融入「古籍批注式」设计，输入框 / 输出框为书卷造型（米白背景 + 赭石棕细边框），光标为朱砂红；解析结果用朱砂红标注重点，贴合古典文本阅读的批注习惯；
视觉装饰：AI 工具区可添加轻量「数字点阵 + 竹简纹路」融合装饰，占比≤5%，突出数字科技感与古典文融合的特色。
二、交互体验精细化要求
基础交互：沿用全局导航、按钮、滚动交互规则，所有按钮（答题、使用工具、提交）为国风圆角样式（圆角 8px），贴合全局交互色彩与动效要求；长页面设置「书卷造型回到顶部按钮」。
题库交互：
筛选交互：筛选按钮点击反馈清晰，多条件筛选结果实时刷新，加载时添加「竹简纹路旋转」水墨动画；
答题交互：点击答题入口平滑跳转到习题页面，习题页面为古籍排版样式（米白背景、墨灰文字、朱砂红答题区域），支持答案输入 / 提交，提交按钮为国风样式；
判分反馈：题库答案自动判分（若有）后，显示国风反馈弹窗，正确答案用赭石棕标注，错误答案用朱砂红标注，附带简单解析。
AI 工具交互：
工具使用：点击工具入口平滑展开操作界面，界面无卡顿，输入框支持 placeholder 提示（国风文案，如 “请输入需解析的古典文本内容”）；
解析交互：点击解析按钮后，添加「毛笔书写」轻量动效，解析结果水墨淡入展示，支持结果复制 / 下载，复制 / 下载按钮为国风线性图标；
反馈交互：解析完成 / 复制成功后，显示国风轻量弹窗，3 秒后水墨淡隐；解析失败时，朱砂红文字提示原因，无刺耳提示音。
三、内容呈现与技术要求
题库内容：
习题名称突出文本名与题型，关键词用赭石棕加粗；
习题解析贴合古典文本专业要求，语言严谨，解析重点用朱砂红标注；
题型 / 难度标注统一，视觉规范，方便用户快速筛选。
AI 工具内容：
工具简介简洁明了，控制在 60 字内，突出核心功能（如文本解析、词句注释、人物关系分析）；
操作提示为国风文案，贴合项目风格，如 “请粘贴古典文本，点击解析获取 AI 批注”。
性能与适配：
AI 工具界面适配多终端，移动端输入框 / 输出框自适应屏幕，操作按钮大小适配手指点击；
页面加载时优先加载题库模块，AI 工具模块懒加载，保证首屏加载速度（PC≤2 秒，移动端≤3 秒）；
所有输入框 / 输出框支持字号调节（小 / 中 / 大），调节后布局自适应，不出现文字溢出 / 错位。
